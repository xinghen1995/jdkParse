##Observable
<ol>
<li>这是观察者模式中的主题，它是一个抽象类</li>
<li>观察者被通知的顺序，默认的实现是依据注册顺序，但子类可能会更改器通知顺序</li>
<li>如果观察者的<tt>equals</tt>返回相同，那么它们被认为是相同的观察者</li>
<li>利用<tt>contain</tt>保证观察者队列中没有重复的元素，可以认为是集合set，但真实的实现是用<tt>Vector</tt></li>
<li>只有当changed为true的时候才通知观察者，这样做是为了防止更新过于灵敏，而导致的频繁通知</li>
<li>在<tt>addObserver</tt>和<tt>deleteObserver</tt>中只对方法进行了同步，而<tt>notifyObservers</tt>则对整个对象进行了同步，这么
做事有意义的，一是为了防止新注册的观察者没有得到通知，而是为了防止未注册的观察者错误的得到消息</li>
</ol>
